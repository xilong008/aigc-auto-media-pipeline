# 小红书爆款文案生成 SOP（数据驱动版）

基于全网高赞爆款笔记的数据拆解，真正的“爆款”绝不是空洞的观点堆砌，而是**“犀利观点 + 扎实论据 + 情绪共鸣”**的结合体。为了解决大模型容易“假大空”、“只讲道理不给证据”的问题，特制定以下文案生成底层 SOP。

---

## 核心底层逻辑：用事实替代形容词
*   ❌ **错误示范**：“老房改造有很多隐形坑，大家一定要注意水电问题，不然会花很多冤枉钱。”（全是抽象概念，毫无说服力）
*   ✅ **正确示范**：“入行十五年，我见过最惨的业主，因为没提前排查老管线老化，入住半年水管爆裂，刚铺的300元/平的实木地板全泡汤，硬生生赔了8万块。”（具体数字 + 痛点场景 + 真实惨案 = 极强说服力）

---

## 爆款结构公式 (AIDA 模型定制版)

### 1. Attention (抓眼球)：场景切入与制造焦虑 (占15%)
*   **动作**：开篇第一句决不能是废话问候，必须直接甩出一个**高频痛点场景**或**颠覆常识的结论**。
*   **要求**：必须带出具体人群或具体困境。
*   *示例*：“很多第一次装修的人以为，选了半包就能省钱，结果最后结算时，增项比报价单高了整整40%。”

### 2. Interest (引兴趣)：抛出具体论据与惨痛案例 (占30%)
*   **动作**：提出问题后，立刻用**真实案例、量化数据或极其细节的实操坑点**来佐证你的观点。
*   **要求**：必须要有“数字（如3万块、40%、2mm的缝隙）”或“行业内幕专用词（如：腻子找平、水电走顶）”，建立专家权威感。
*   *示例*：“别不信，我上周刚去一套80平的老公房现场，业主为了省2000块钱没做墙面冲筋找平，现在装上定制柜，侧边留了一道两指宽的黑缝，天天看着堵心。”

### 3. Desire (强渴望)：给出极具落地性的解决方案 (占40%)
*   **动作**：指出问题后，提供**“拿来就能用”**的明确方法论。
*   **要求**：方案不能是大方向（比如“你要找靠谱的人”），必须是具体动作（比如“签合同必须加一句：闭口合同，增项超过5%由施工方承担”）。
*   *示例*：“记住这三条铁律：1. 拆旧时必须要求原底盒彻底清理；2. 水电改造明确要求活线铺设，拍照留存；3. 墙面找平合同里必须写明顺平还是垂平。”

### 4. Action (促行动)：极强的私域引流与互动钩子 (占15%)
*   **动作**：在读者情绪最高涨时，利用“专家身份”提供一对一帮助，留下诱饵。
*   **要求**：自然且真诚的呼吁。
*   *示例*：“老房翻新是一场排雷战。如果你也正准备改造，拿不准报价单有没有水分，在评论区留下【你的面积+坐标】，我帮你避开那些最容易踩的坑。”

---

## AI Prompt 优化校验清单 (Checklist)

大模型在输出正文时，必须经过以下校验机制：
1. [ ] **论据浓度校验**：全文是否至少包含 2 个具体的数字或真实案例？
2. [ ] **行业深度校验**：是否使用了 2-3 个只有业内人士才知道的硬核专业术语？
3. [ ] **情绪价值校验**：是否有“别不信”、“听我一句劝”、“我见过最惨的”这种拉近距离的口语化引导？
4. [ ] **降维打击校验**：排版是否清爽？是否使用了 Emoji 作为视觉引导？短句是否多于长难句？

> 此 SOP 应当硬编码注入到大模型的 System Prompt 中，强制大模型在生成正文（`body`）时，严格遵循“提出观点 -> 给出现实惨案/数据 -> 给出避坑实操”的三段式结构。
